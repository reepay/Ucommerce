How to setup the Reepay App

After the app is installed into Ucommerce you need to create a payment method by using the Reepay service.

This follows the standard way of setting up a payment method within Ucommerce.

ReepayPrivateKey is required for the payment method to function.

You can find the ReepayPrivateKey in Reepay's system. Go to https://admin.reepay.com/ and login. Navigate to Developers -> Api credentials in the left panel

Accept_url (required): The URL for the user to be redirected to, once the payment is completed.

Accept_url_change_cc (optional): The URL for the user to be redirected to after changing his credit card information.

Error_url (required): The URL for the user to be redirected to, if the payment fails.

Cancel_url (required): The URL for the user to be redirected to after clicking Cancel link on the Reepay payment window (usually it is a Checkout page).

Cancel_url_my_account (optional): The URL for the user to be redirected to after clicking Cancel link on the Reepay payment window when the payment is only for subscription (usually it is a MyAccount page from where a member can buy subscription).

Button_text_for_subscription (optional): The text that will be displayed on the submit button of reepay payment window when subscription is going to be bought. Must be not longer than 32 characters including spaces. If no value is provided, the Reepay default value ("Gem" for Danish sites) will be displayed.

Test (required): Must be true on test sites and false on production

How to automatically capture, refund and cancel payments

Ucommerce can automatically capture, refund and cancel payments. If ShouldSubscriptionBeCreated and SubscriptionAmount order properties were set before the payment was authorised, please don't delete or change them as they must be used during capture, refund and cancel payments (see Order properties section for more information regarding order properties).
